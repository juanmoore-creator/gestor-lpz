TECNOLOGÍAS Y FUNCIONALIDADES DEL PROYECTO

## FRONTEND

1.  **React (v19)**
    -   Uso: Biblioteca principal de JavaScript para construir la interfaz de usuario.
    -   Función: Permite crear componentes reutilizables y gestionar el estado de la aplicación.

2.  **Vite**
    -   Uso: Herramienta de compilación y servidor de desarrollo.
    -   Función: Proporciona un entorno de desarrollo rápido y optimiza la construcción final para producción.

3.  **TypeScript**
    -   Uso: Lenguaje de programación.
    -   Función: Añade tipado estático a JavaScript para prevenir errores y mejorar la autocompletado.

4.  **Tailwind CSS**
    -   Uso: Framework de CSS para estilizado rápido.
    -   Función: Facilita el diseño mediante clases utilitarias directamente en el HTML/JSX.
    -   Complementos: `clsx` y `tailwind-merge` para gestionar clases condicionales y conflictos.

5.  **React Router (react-router-dom)**
    -   Uso: Enrutamiento del lado del cliente.
    -   Función: Gestiona la navegación entre las páginas de la aplicación sin recargar el navegador.

6.  **Framer Motion**
    -   Uso: Biblioteca de animaciones.
    -   Función: Permite crear transiciones y animaciones fluidas en componentes React.

7.  **Recharts**
    -   Uso: Visualización de datos.
    -   Función: Creación de gráficos estadísticos y visualizaciones de datos.

8.  **Google Maps API Wrappers**
    -   Uso: Integración de mapas.
    -   Función: Se utiliza (`@react-google-maps/api`) para mostrar mapas interactivos, como en la previsualización de ubicaciones de propiedades.

9.  **Radix UI**
    -   Uso: Componentes de UI.
    -   Función: Proporciona primitivas de componentes accesibles y no estilizadas, como las pestañas (`@radix-ui/react-tabs`).

10. **Lucide React**
    -   Uso: Iconografía.
    -   Función: Colección de iconos vectoriales ligeros y consistentes.

11. **date-fns**
    -   Uso: Manipulación de fechas.
    -   Función: Librería avanzada para parsear, formatear y operar con fechas y horas.

## GENERACIÓN DE DOCUMENTOS Y DATOS

12. **html2canvas & jspdf**
    -   Uso: Generación de PDFs a partir de HTML.
    -   Función: `html2canvas` captura componentes como imágenes y `jspdf` los integra en archivos PDF para crear reportes.

13. **PapaParse**
    -   Uso: Procesamiento de archivos CSV.
    -   Función: Permite analizar datos CSV directamente en el navegador, usado para importaciones.

## BACKEND (API Serverless)

14. **Node.js**
    -   Uso: Entorno de ejecución para el backend.
    -   Función: Se utiliza para crear las funciones serverless (en la carpeta `api/`) que manejan la lógica de negocio segura.

15. **Firebase (Cliente y Admin SDK)**
    -   Uso: Plataforma BaaS (Backend as a Service).
    -   Función: Se usa Firestore como base de datos en tiempo real (cliente) y el **Admin SDK** en el backend para operaciones con privilegios.

16. **Google APIs (googleapis)**
    -   Uso: Integración con servicios de Google.
    -   Función: El cliente de Node.js se usa en el backend para autenticar y interactuar con **Google Calendar** (agendar visitas) y **Google Drive** (subir archivos).

17. **ImageKit (Cliente y SDK de Node.js)**
    -   Uso: Gestión de activos digitales (imágenes).
    -   Función: Se usa el SDK de cliente para subir imágenes y el de Node.js para la autenticación segura en el backend.

18. **Integración con WhatsApp**
    -   Uso: Comunicación con clientes.
    -   Función: El backend (`api/whatsapp/`) está preparado para enviar mensajes y recibir notificaciones (webhooks) a través de la API de WhatsApp.

19. **Formidable**
    -   Uso: Procesamiento de formularios en Node.js.
    -   Función: Permite gestionar la subida de archivos en las funciones del backend.

## CALIDAD DE CÓDIGO Y DEPLOYMENT

20. **ESLint**
    -   Uso: Linter de código.
    -   Función: Analiza el código para encontrar problemas, asegurar consistencia y aplicar buenas prácticas.

21. **Vite PWA (Progressive Web App)**
    -   Uso: Mejora de la aplicación web.
    -   Función: El plugin `vite-plugin-pwa` configura la aplicación para que pueda ser "instalada" en dispositivos y funcionar sin conexión.
